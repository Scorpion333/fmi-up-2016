#Структури и динамична памет

###Задача 0.
Напишете структура Programmer, която има следните член-данни:
- име - символен низ до 20 символа
- брой езици, които знае - цяло число
- заплата - цяло число

Напишете следните методи за Programmer:
- void read() - прочита член-данните от клавиатурата.
- void print() - извежда на екрана член-данните
- bool correctData() - проверява дали член-данните са коректни - т.е. числата са положителни
- bool knowsMoreThan(Programmer other) - проверява дали знае повече езици от друг програмист 

###Задача 1.
Напишете програма, която прочита число n, след което прочита масив от n цели числа и ги извежда сортирани по дължина, т.е.
най-отпред са едноцифрените (без значение в какъв ред), после са двуцифрените и т.н.

###Задача 2.
Напишете функция strcpy_better(char*& a, char* b), която приема два указателя към символни низове и прави така,
че първият низ да стане същия като втория. Вторият низ може да има произволна дължина.

###Задача 3.
Напишете програма, която прочита число n, след което прочита данните за n програмисти и проверява дали данните са коректни.
Да се изведат "коректните" програмисти, сортирани по броя езици, които знаят.

###Задача 4.
Напишете структура Game със следните член-данни:
- име - указател към символен низ с произволна дължина.
- резултати от играта - масив от произволен брой цели числа
- брой резултати - цяло число, в което ще пазим размера на масива с резултатите
- максимален брой резултати - цяло число, равно на лимита за броя резултати, които могат да се запишат

Напишете следните методи за Game:

- void setName(char* newName) - дава име на играта
- void setMaxResults() - дава лимит за резултатите
- void saveResults(int* newResults, int size) - добавя числа в масива с резултатите, ако има място
- int highScore() - връща рекорда, а ако няма записани резултати - 0
- void clearMemory() - освобождава заделената памет

###Задача 5.
Добавете член-данна game на Programmer - игра, която е написана от програмиста.
Напишете програма, която прочита данни за произволен брой програмисти и извежда името на програмиста, чиято игра можа да се играе най-много,
т.е. има най-голям лимит за резултатите.
